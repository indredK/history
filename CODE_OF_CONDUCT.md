# 社区行为准则 (Contributor Covenant Code of Conduct)

## 我们的承诺

为了促进一个开放和欢迎的环境，我们作为贡献者和维护者承诺，无论年龄、体型、残疾、种族、性别认同与表现、经验水平、国籍、个人风格、宗教信仰或性别身份和性取向，参与者在我们的项目和社区中应获得无骚扰的体验。

## 我们的标准

### 建设性行为示例

- 使用欢迎且包容的语言
- 尊重不同的观点与经验
- 优雅地接受建设性批评
- 关注对社区最有益的事
- 表现出对他人的同情

### 不接受的行为

不可接受的行为包括但不限于：

- 使用性化的语言或意象，以及不受欢迎的性骚扰
- 人身攻击，贬低或诋毁性评论
- 公开或私下骚扰
- 未经明确许可发布他人隐私信息（例如住址或电子邮件）
- 其他不专业的行为
- 违反学术诚信或数据完整性的行为

## 执行

### 举报程序

项目维护者负责澄清可接受行为的标准，并对不可接受的行为采取适当的纠正措施。

如果你发现了不符合本行为准则的行为，请通过以下任一方式报告：

1. **GitHub Issues/Discussion**: 在公开讨论中礼貌地指出不当行为（如果涉及敏感话题，可以私密讨论）
2. **直接联系维护者**: 向项目主要维护者发送私信或电子邮件
3. **邮件**: 将详细情况发送至 `maintainers@example.com`（建设中）

**举报应包括**:
- 不当行为的具体描述
- 何时发生（如适用）
- 涉及的人员（如果可以安全地分享）
- 任何相关证据（如消息截图、讨论链接）
- 你建议的解决方案（可选）

### 调查与处理

1. **初步确认**: 维护者将在 3 个工作日内确认收到报告
2. **调查**: 我们将进行充分的调查，可能需要向相关方了解情况
3. **决定**: 根据事实和本准则，做出相应的处理决定
4. **通知**: 将处理结果通知报告者与被指控方（如适当）
5. **上诉**: 任何方都可以在决定后 7 天内提出上诉

### 处罚措施

根据违规的严重程度，可能的处罚包括：

1. **警告**: 对初次轻微违规，维护者将直接警告参与者
2. **评论移除**: 删除违规的评论或内容
3. **临时禁言**: 禁止在指定期间参与项目活动（通常 7-30 天）
4. **永久禁言**: 对于严重或持续违规，永久开除该贡献者

## 范围

本行为准则适用于：

- 项目仓库（Issues、PR、Discussion、Commit 评论）
- 项目相关的外部交流渠道（Discord、微信群、线下活动等），只要行为与项目相关联

对于发生在项目外但影响项目社区安全和包容性的行为，维护者也可能采取行动。

## 特殊情况说明

### 学术诚实

本项目处理历史与地理学术数据。我们特别重视学术诚实：

- 数据来源必须真实、可验证
- 不接受捏造、篡改或冒用他人研究成果
- 对学术争议保持开放态度，但需要以证据为基础进行讨论

### 语言包容性

虽然项目主要使用中文和英文，但我们欢迎其他语言的参与者。请注意：

- 翻译或用非主要语言交流时，你的努力将受到感谢与尊重
- 如果存在语言障碍，维护者将尽量帮助理解

### 知识产权

所有贡献必须尊重知识产权法：

- 不上传受版权保护的内容（除非你拥有或获得许可）
- 对于使用的第三方资源，必须清晰标注来源与许可
- 数据贡献遵循 CC-BY-4.0，代码遵循 MIT

## 维护者指南

### 维护者应该

- 遵守本行为准则，树立表率
- 及时、公正地处理举报
- 保护举报人的隐私
- 定期审查和更新准则
- 在社区中积极推广包容文化

### 维护者不应该

- 因个人喜好而做出报复性处罚
- 对举报置之不理
- 要求举报人进行公开道歉（除非他们同意）

### 常见问题

**Q: 我不小心违反了准则，会被永久封禁吗？**  
A: 不一定。首次轻微违规通常会收到警告。我们相信人会犯错，鼓励通过对话和改正来解决问题。

**Q: 我想举报但担心报复怎么办？**  
A: 举报可以私密进行。维护者会采取措施保护举报人不受报复。

**Q: 什么样的反对意见被视为不可接受？**  
A: 反对观点和建设性批评是受欢迎的。不可接受的是使用侮辱、人身攻击或骚扰性语言来表达反对。

**Q: 如果我不同意处罚决定怎么办？**  
A: 你可以在 7 天内向主要维护者发起上诉，请求重新审视。
